# Simple Book Recommendation System (Spark + Flask)

This project is a **very simple end‑to‑end book recommendation system** that runs locally on the machine. It uses:

- **Apache Spark (PySpark)** for training an **ALS (matrix factorization)** recommendation model.
- **Flask** for a tiny web frontend to query recommendations.
- Either a small demo CSV or the **Kaggle Book‑Crossing dataset** (via a prep script) as the source of user–book ratings.

---

## 2. Data flow and model flow

**Input data (`book_ratings.csv`)**

- Columns: `userId, bookId, rating, title`
- Example rows:
  - `1,101,5,Harry Potter`
  - `1,102,4,The Hobbit`

**Processing steps (Spark / ALS)**

1. `src/train_als.py` starts a local Spark session.
2. Loads `data/book_ratings.csv` with header and infers schema.
3. Renames columns to the types expected by ALS:
   - `userId` → `user_id` (int)
   - `bookId` → `book_id` (int)
   - `rating` → `rating` (float)
4. Splits the data into **train (80%)** and **test (20%)**.
5. Trains an **ALS** model with:
   - `userCol="user_id"`
   - `itemCol="book_id"`
   - `ratingCol="rating"`
   - `rank=20`, `maxIter=10`, `regParam=0.1`
6. Evaluates the model using **RMSE** on the test set and prints the score.
7. Generates **top‑10 recommendations per user** using the trained model.
8. Joins the recommendations with `title` from the original CSV.
9. Writes a flat file: `data/user_recommendations.csv`.

**Output data (`user_recommendations.csv`)**

- Columns (example):
  - `user_id`: numeric user ID (e.g. 1)
  - `book_id`: numeric book ID (e.g. 101)
  - `score`: predicted rating/relevance from ALS
  - `title`: book title (e.g. "Harry Potter")

This file is what the Flask app reads to serve recommendations.

## 3. Frontend / API flow (Flask + HTML)

**Backend (`src/app.py`)**

1. On startup, `app.py`:
   - Locates `data/user_recommendations.csv`.
   - Loads it into a **pandas DataFrame** for fast lookup.
2. Route `/` (GET):
   - Renders `templates/index.html` with an empty results table.
3. Route `/recommend` (POST):
   - Reads the `user_id` from the HTML form.
   - Validates that it is a numeric ID.
   - Filters the DataFrame for that `user_id` and sorts by `score` (descending).
   - Passes the resulting rows (list of dicts) to the template.

**Frontend (`templates/index.html`)**

- Very simple and clean page:
  - A single text box for **User ID**.
  - A button: **“Get recommendations”**.
  - If there is an error (invalid user or no results), it shows a short message.
  - Otherwise it displays a table:
    - Rank (1, 2, 3, ...)
    - `Title`
    - `Book ID`
    - `Score` (model prediction)

## 4. How to run the project locally

### 4.1. Prerequisites

- Python 3.9+ recommended.
- Java (for Spark) installed and available on your PATH.

### 4.2. Install Python dependencies

From the project root:

```bash
pip install -r requirements.txt
```

### 4.3. Prepare the data

Place the CSV file in `data/`:

```bash
mkdir -p data
cp /path/to/book_ratings.csv data/book_ratings.csv
```

Make sure the header is:

```text
userId,bookId,rating,title
```

### 4.4. Train the model and generate recommendations

Run the Spark training script:

```bash
python src/train_als.py
```

This will:

- Start Spark in local mode.
- Train the ALS model.
- Print the test RMSE.
- Save recommendations to `data/user_recommendations.csv`.

### 4.5. Start the local web app

Run the Flask app:

```bash
python src/app.py
```

Then open the browser at:

```text
http://127.0.0.1:5000/
```

Type a valid **user ID** from your dataset (for example, `1`, `2`, `3`, etc.) and click **“Get recommendations”**.

A simple table of recommended books for that user will be visible.

1. Show the **raw data** file `data/book_ratings.csv`.
2. Run `python src/train_als.py` and explain:
   - How we train ALS.
   - How we evaluate with RMSE.
   - How we generate a top‑N list of recommendations per user.
3. Open `data/user_recommendations.csv` to show the predictions.
4. Start the Flask app with `python src/app.py`.
5. In the browser:
   - Enter a user ID that exists in your dataset.
   - Show the top recommended books and their scores.

This gives a **complete, local, end‑to‑end recommendation system**: from raw CSV data, to model training, to a small UI that a user can interact with.
